# Contributing

## Why This Project Exists

Current AI tools introduced into the design process carry real risks that are often overlooked. This project was built specifically to address five documented concerns:

- **Limits to Divergent Thinking** — AI tools that generate polished outputs too early can collapse the exploration space and push designers toward the first plausible solution rather than the best one. This project keeps the AI in a supporting, not leading, role during ideation.

- **False Validation** — AI responses can feel authoritative even when they are wrong or misaligned with the user's intent. This project emphasizes human judgment at every phase and avoids framing AI output as final or correct.

- **Lack of Prototyping** — Many AI design tools skip the messy middle of the design process entirely. This project integrates whiteboard-based prototyping and artifact-grounded workflows to keep physical and spatial thinking central.

- **Linear Workflow Tension** — Design is non-linear, but most AI tools force a prompt-response loop that implies a straight path forward. This project uses phase modes (Ideation, Conceptualization, Execution) to reflect how design actually moves.

- **Unhealthy Cognitive Offloading** — Overreliance on AI for judgment, decisions, and critique degrades the designer's own capabilities over time. This project is built around the designer remaining the primary thinker, using AI as a communication and documentation aid.

## Contribution Guidelines

Contributions should support one or more of these principles:

- Human-led creative control
- Multimodal design communication
- Phase-aware design process support
- Transparent, inspectable AI behavior

If a contribution makes the AI more autonomous, more generative by default, or harder for the designer to override, it is likely out of scope.

Please keep API credentials out of the repository and use environment variables only.
